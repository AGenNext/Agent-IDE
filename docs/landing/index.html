Quickstart

IDE + backend running in under a minute.

1

Clone & install

Requires Node 22 and Yarn (via corepack).

2

Build the workspace

Single command builds both the Theia frontend and the backend.

3

Start IDE + backend

Theia on :3000, backend API + WebSocket on :3001.

4

Run your first agent

Open the Runs panel, hit ⚡ Live Run, pick a model and task.


Terminal
# 1. Clone
git clone https://github.com/agennext/agent-ide
cd agent-ide

# 2. Install
corepack enable
yarn install

# 3. Build
yarn build

# 4. Start the Theia IDE (port 3000)
yarn start

# 5. Start the backend (port 3001) — separate terminal
cd packages/agent-ide-backend
npx ts-node src/server.ts

.env — backend environment
# LLM — at least one required for live mode
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...

# Tools
ALLOW_SHELL=true          # enables shell + code_exec tools
BRAVE_API_KEY=...         # enables web_search tool
DATABASE_URL=postgres://... # enables db_query tool

# MCP
GITHUB_TOKEN=ghp_...      # enables GitHub MCP server

# Auth (optional — demo mode works without)
AUTH_ENABLED=true
JWT_SECRET=change-in-production
DEMO_PASSWORD=yourpassword

Docker
docker build -t agennext/agent-ide:local .
docker run --rm -p 3000:3000 -p 3001:3001 \
  -e OPENAI_API_KEY=sk-... \
  -e ALLOW_SHELL=true \
  agennext/agent-ide:local

MCP Servers

Connect any MCP (Model Context Protocol) server over stdio. Tools are auto-discovered and surfaced to the agent loop as mcp__<serverId>__<toolName>.

ServerCommandRequires
filesystemnpx -y @modelcontextprotocol/server-filesystem .—
brave-searchnpx -y @modelcontextprotocol/server-brave-searchBRAVE_API_KEY
postgresnpx -y @modelcontextprotocol/server-postgres $DATABASE_URLDATABASE_URL
puppeteernpx -y @modelcontextprotocol/server-puppeteer—
githubnpx -y @modelcontextprotocol/server-githubGITHUB_TOKEN

Connect a server
POST /api/mcp/servers/github/connect

List connected tools
GET /api/mcp/tools
Add a custom server
POST /api/mcp/servers
{
  "id": "my-server",
  "name": "My MCP Server",
  "transport": "stdio",
  "command": "npx -y my-mcp-package",
  "env": { "API_KEY": "${MY_API_KEY}" }
}

Server config is persisted to .mcp.json in your workspace root. Env var references like ${GITHUB_TOKEN} are resolved at connect time from the process environment.

Submit a run and stream steps
# 1. Submit
POST http://localhost:3001/api/runs
{
  "model": "claude-sonnet-4-6",
  "task": "Summarise the top 3 open GitHub issues in this repo",
  "tools": ["vector_search", "http_client"],
  "apiKey": "sk-ant-..."
}

# Response → { "runId": "abc123", "wsUrl": "/ws/abc123" }

# 2. Stream via WebSocket
wscat -c ws://localhost:3001/ws/abc123

WebSocket message types
run:started          → run record
run:step             → TraceStepRecord { type: thought | action | observation | result | error }
run:completed        → final run record with cost/token totals
run:failed           → { error: string }
governance:approval-request   → pending tool call details
governance:approval-resolved  → approved | rejected

Frontend SDK

backend-client.ts — a typed HTTP + WebSocket client shipped inside the Theia extension. Import it in any panel or custom extension.

Import
import { backendClient } from '../runtime/backend-client';

Submit a run + stream steps
// Submit
const { runId } = await backendClient.submitRun({
  model: 'claude-sonnet-4-6',
  task:  'Analyse this codebase',
  tools: ['repo_index', 'vector_search'],
});

// Stream
backendClient.streamRun(runId, (msg) => {
  if (msg.type === 'run:step') {
    console.log(msg.payload);   // TraceStepRecord
  }
});
Knowledge store
// Ingest text
await backendClient.ingestText(
  'My doc title',
  content
);

// Semantic search
const results = await backendClient.searchKnowledge(
  'agent lifecycle',
  5   // topK
);

MCP tools
await backendClient.connectMcpServer('github');
const tools = await backendClient.listMcpTools();
// → [{ serverId, name, description, inputSchema }]

Auth
const { token, user } = await backendClient.login('you@example.com', 'password');
// Token is persisted automatically via session-store.ts (localStorage)

const me = await backendClient.getMe();
// → { userId, email, name }

Governance approvals
// Poll pending approvals
const pending = await backendClient.listApprovals();

// Approve or reject a held tool call
await backendClient.approveRequest(pending[0].id);
await backendClient.rejectRequest(pending[0].id);
